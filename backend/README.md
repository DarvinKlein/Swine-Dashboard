# Backend — Laravel API

This folder contains the application code (models, controllers, migrations, routes, config)
for the Swine Dashboard API. It is **not a full Laravel skeleton** — you drop these files into
a fresh Laravel install, since the framework itself is best generated by the Laravel installer
(so it always matches your PHP/Composer version).

## Setup

1. Create a fresh Laravel project somewhere outside this folder:

   ```bash
   composer create-project laravel/laravel swine-dashboard-api
   cd swine-dashboard-api
   ```

2. Copy the contents of this `backend/` folder into the new project, merging/overwriting:

   ```bash
   cp -r app/Models/* ../swine-dashboard-api/app/Models/
   cp -r app/Http/Controllers/Api ../swine-dashboard-api/app/Http/Controllers/
   cp -r database/migrations/* ../swine-dashboard-api/database/migrations/
   cp routes/api.php ../swine-dashboard-api/routes/api.php
   cp config/cors.php ../swine-dashboard-api/config/cors.php
   ```

   If `routes/api.php` doesn't exist yet in your Laravel version, run
   `php artisan install:api` first (Laravel 11+) to generate it, then paste in the routes.

3. Create a MySQL database and copy `.env.example` values into your project's `.env`
   (adjust `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` to match your local MySQL):

   ```bash
   mysql -u root -p -e "CREATE DATABASE swine_dashboard;"
   php artisan key:generate
   ```

4. Run the migrations:

   ```bash
   php artisan migrate
   ```

5. Start the API server:

   ```bash
   php artisan serve
   ```

   The API will be available at `http://localhost:8000/api`.

## Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/swines` | List all saved swine records |
| POST | `/api/swines` | Create a swine record (`swine_name`, `breeding_date`) — server computes pregnant/iron/labor dates |
| GET | `/api/fattening-batches` | List all fattening batches with `total_cost` |
| POST | `/api/fattening-batches` | Create a batch (`number_of_heads`) |
| POST | `/api/fattening-batches/{batch}/feeds` | Add a feed cost entry to a batch (`amount`) |

## Notes

- `config/cors.php` allows requests from `http://localhost:5173` (the default Vite dev server port).
  Update `allowed_origins` if you deploy the frontend elsewhere.
- Pregnant/iron/labor dates are computed server-side from `breeding_date` (day 20, day 100, day 114-118)
  so the frontend and database always agree.
- `total_cost` on a fattening batch is the sum of all its feed entries, computed on the fly.
